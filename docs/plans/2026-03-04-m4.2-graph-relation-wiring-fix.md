# M4.2: Graph Relation Wiring Fix

**Date**: 2026-03-04
**Status**: Complete

## Problem

The GraphRAG benchmark showed `graph_expanded: 0-2` across all queries because almost zero graph edges were being wired during ingestion. The full infrastructure was in place (adapter IPC, pipeline aggregation, `wire_pipeline_relations()`), but target file matching failed silently for all cross-file relations.

### Root Cause

`wire_pipeline_relations()` matched `pc.metadata.source_file == rel.to_file`, but:

1. **Rust `crate::` imports**: `to_file` was set to the SOURCE file path (not the target), so every "import" edge pointed back at itself
2. **Rust `super::` imports**: Same problem — `to_file = file_path` regardless of actual target
3. **External crate imports**: `to_file = "std::collections"` — no match (expected, OK)
4. **JS/TS relative imports**: `to_file = "./utils"` — never matches `src/utils.ts`

## Fix

### 1. Emit Resolvable `to_file` from Tree-sitter

**File**: `crates/corvia-adapter-git/src/treesitter.rs`

- `crate::foo::bar` now emits `CRATE_REF:crates/corvia-kernel/src:foo::bar` using `infer_crate_src_root()` to extract the crate's `src/` root
- `super::foo` resolves relative to the source file's parent directory, also emitting `CRATE_REF` format
- Fallback to `file_path` when resolution is impossible (e.g., `super` from crate root)
- Added `infer_crate_src_root()` helper

### 2. Multi-Strategy Target Resolution in `wire_pipeline_relations()`

**File**: `crates/corvia-cli/src/main.rs`

Rewrote with `HashMap`-based file index and 4 resolution strategies:

1. **Direct match**: `source_file == to_file` (handles "implements"/"contains" within same file)
2. **CRATE_REF resolution**: `CRATE_REF:<root>:<module>` → try `<root>/<module>.rs` and `<root>/<module>/mod.rs`
3. **JS/TS relative**: `./utils` → resolve relative to source file, try `.ts/.js/.tsx/.jsx` extensions
4. **Suffix fallback**: match `to_file` as a suffix of known `source_file` paths

Also added self-edge filtering for "imports" relations.

## Benchmark Results

| Metric | Before | After |
|--------|--------|-------|
| Graph relations stored (ingest) | ~0 | **552** |
| Graph edges (rebuilt) | ~0 | **324** |
| Q3 graph_expanded | 2 | **6** |
| Q4 graph_expanded | 1 | 1 |

- Q1/Q2 remain at 0 because those queries match docs/config files with no code-level graph edges (expected behavior)
- Q3 (cross-file code tracing) showed 3x improvement, confirming the wiring fix works
