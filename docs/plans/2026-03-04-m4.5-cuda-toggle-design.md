# M4.5: Toggleable CUDA Support for corvia-workspace

**Date:** 2026-03-04
**Status:** Approved
**Scope:** Full GPU stack — embeddings (ONNX/fastembed), future chat models (candle), future vLLM

## Problem

`corvia-inference` runs ONNX embedding inference on CPU. Users with NVIDIA GPUs get no acceleration. The devcontainer has no GPU awareness. CUDA setup should be optional and toggleable, matching the existing `corvia-workspace enable/disable` pattern for Ollama and SurrealDB.

## Approach: Script-Only Toggle

Extend `corvia-workspace.sh` with a `cuda` service. No Dockerfile changes. CUDA toolkit installed at enable-time, CUDA-enabled binary downloaded from GitHub releases.

## Toggle Command Interface

```
corvia-workspace enable cuda     # Install CUDA, download GPU binary
corvia-workspace disable cuda    # Revert to CPU binary
corvia-workspace status          # Shows cuda row with GPU info
```

### Enable Flow

1. Check `nvidia-smi` is accessible (fail with guidance if not)
2. Install CUDA toolkit + cuDNN via apt if not present
3. Download `corvia-inference-cuda-linux-amd64` from GitHub releases
4. Replace `/usr/local/bin/corvia-inference` (backup CPU version as `corvia-inference-cpu`)
5. Set flag `cuda=enabled`
6. Restart `corvia-inference` if running

### Disable Flow

1. Restore CPU binary from backup
2. Set flag `cuda=disabled`
3. Restart `corvia-inference` if running

## GPU Detection & Prerequisites

```bash
# 1. Check nvidia-smi (proves GPU driver + hardware present)
nvidia-smi --query-gpu=name,driver_version,memory.total --format=csv,noheader

# 2. Check CUDA toolkit (install if missing)
nvcc --version || install_cuda_toolkit

# 3. Verify libcudnn (install if missing)
ldconfig -p | grep libcudnn || install_cudnn
```

### CUDA Installation

Uses NVIDIA's apt repository for Debian Trixie (matching the `rust:1.88-trixie` base image). Installs `cuda-toolkit-12-x` and `libcudnn9-dev`. Approximately 2GB download.

### Failure Modes

| Condition | Message |
|-----------|---------|
| No `nvidia-smi` | "No NVIDIA GPU detected. Ensure NVIDIA drivers are installed on the host and GPU passthrough is configured." |
| WSL2 without GPU | Check `/usr/lib/wsl/lib/libcuda.so` exists |
| No network | "Cannot download CUDA toolkit. Check internet connectivity." |
| Download fails | Fall back to source build with `--features cuda` |

## Binary Management

```
/usr/local/bin/
  corvia-inference          # Active binary (CPU or CUDA)
  corvia-inference-cpu      # Backup of CPU binary (created on first enable)
  corvia-inference-cuda     # Downloaded CUDA binary
```

### Download URL

```
https://github.com/chunzhe10/corvia/releases/latest/download/corvia-inference-cuda-linux-amd64
```

Requires a CI matrix addition to build the CUDA variant. Out of scope for this design — the script falls back to source build if download fails.

## Status Display

Extends `show_status()`:

```
=== Corvia Workspace Services ===

  ollama       enabled=disabled running=no
  surrealdb    enabled=disabled running=no
  cuda         enabled=enabled  gpu=NVIDIA RTX 4090 (24GB) driver=555.42
  corvia       enabled=always   running=yes

  Config: provider=corvia model=nomic-embed-text-v1.5 store=lite
```

When CUDA is disabled or no GPU:
```
  cuda         enabled=disabled gpu=not detected
```

## Post-Start Integration

No special restart logic needed. The binary choice is persistent. Enable/disable scripts kill and restart `corvia-inference` if it's running.

`post-start.sh` already re-enables previously enabled services via the flags file. CUDA flag (`cuda=enabled`) is checked — if set, the CUDA binary is already in place, so no action needed beyond the normal inference server start.

## Rebuild Integration

`corvia-workspace rebuild` gains CUDA awareness:

```bash
rebuild_binaries() {
    if [ "$(get_flag cuda)" = "enabled" ]; then
        cargo install --path crates/corvia-inference --features cuda
    else
        cargo install --path crates/corvia-inference
    fi
}
```

## Cargo Feature Gate

```toml
# crates/corvia-inference/Cargo.toml
[features]
default = []
cuda = ["fastembed/cuda"]  # Enables ONNX Runtime CUDA EP
```

No Rust code changes required. `fastembed` automatically uses the CUDA execution provider when compiled with the feature and a GPU is available, falling back to CPU otherwise.

## Future: vLLM and Full GPU Stack

The `cuda` toggle establishes the GPU foundation:

- **`corvia-workspace enable vllm`** — future toggle that spins up a vLLM container with `--gpus all` (requires NVIDIA Container Toolkit). The `cuda` flag serves as a prerequisite.
- **Chat model GPU acceleration** — candle's CUDA features can be gated similarly with a `cuda` cargo feature on corvia-inference.
- **NVIDIA Container Toolkit detection** — added to status output when Docker GPU services are introduced.

## Non-Goals

- Changing the Dockerfile or devcontainer.json (no rebuild required)
- CI pipeline for CUDA binary builds (prerequisite, separate work)
- vLLM integration (future M5+ work)
- Windows/macOS GPU support (Linux/WSL2 only)
