# M3.3: Embedding & Chunking Strategy Design

**Date:** 2026-03-02
**Status:** Shipped (v0.3.0)
**Depends on:** None (can build in parallel with M3.1/M3.2; shares TokenEstimator with D64)
**Decisions:** D65 (chunking architecture), D66 (format routing), D67 (sibling merging), D68 (scope), D69 (trait clean break)

## Summary

Add a trait-based chunking framework to `corvia-kernel` that standardizes how all content is split, merged, and prepared for embedding. A `ChunkingStrategy` trait lets adapters provide domain-specific chunking logic (AST for code, heading-based for markdown) while the kernel's `ChunkingPipeline` enforces universal quality concerns: token budget limits, oversized chunk splitting, small chunk merging, and overlap context. A `FormatRegistry` routes file formats to strategies with hybrid auto-detect + adapter overrides.

## Motivation

1. **Silent truncation.** Embedding engines hard-cap at 4000 chars (`MAX_EMBED_CHARS`). Large functions or long documents get silently truncated — lost information with no feedback.

2. **Prose is chunked as whole-files.** Markdown, TOML, YAML, JSON, and PDF files go through as single chunks. A 50-page README or a long `Cargo.toml` becomes one embedding, destroying retrieval granularity.

3. **No universal enforcement.** Token limits, overlap, and merging are universal concerns but currently don't exist anywhere. Each adapter would need to independently implement them — duplicated, unenforced, error-prone.

4. **Multi-format repos are the norm.** A single Git repo contains code, docs, configs, and sometimes PDFs. Each format needs different chunking semantics, but all need the same quality guarantees.

5. **Benchmarking readiness.** Trait-based strategies let M6 eval swap chunking approaches and measure downstream retrieval quality without code changes.

## Architecture

```
┌──────────────────────────────────────────────────────────┐
│                    ChunkingPipeline                       │
│                   (kernel orchestrator)                    │
│                                                           │
│  1. Format detect → route to strategy                     │
│  2. strategy.chunk() → raw chunks                         │
│  3. strategy.merge_small() OR kernel default merge        │
│  4. Token budget check                                    │
│  5. strategy.split_oversized() for budget violations      │
│  6. strategy.overlap_context() between final chunks       │
│  7. Emit metrics → return Vec<ProcessedChunk>             │
└───────────────┬─────────────────────────────────────┬─────┘
                │                                     │
     ┌──────────▼──────────┐             ┌────────────▼────────┐
     │  ChunkingStrategy   │             │   FormatRegistry     │
     │     (trait)          │             │                      │
     │                      │             │  .rs  → AstChunker   │
     │  chunk()             │             │  .md  → MarkdownChkr │
     │  split_oversized()   │  ◄───────  │  .toml → ConfigChkr  │
     │  merge_small()       │   selects  │  .pdf → PdfChunker   │
     │  overlap_context()   │             │  *   → FallbackChkr  │
     └─────────────────────┘             └──────────────────────┘
```

## Trait Definitions

### ChunkingStrategy

```rust
pub trait ChunkingStrategy: Send + Sync {
    /// Identifier for metrics attribution and benchmarking.
    fn name(&self) -> &str;

    /// Supported file extensions this strategy handles.
    fn supported_extensions(&self) -> &[&str];

    /// Domain-specific: extract semantic boundaries from source.
    /// Returns raw chunks WITHOUT embeddings.
    fn chunk(
        &self,
        source: &str,
        meta: &SourceMetadata,
    ) -> Result<Vec<RawChunk>>;

    /// Domain-specific: split an oversized chunk that exceeds token budget.
    /// Called by kernel when a chunk violates the budget.
    /// Default: split at character midpoint.
    fn split_oversized(
        &self,
        chunk: &RawChunk,
        max_tokens: usize,
    ) -> Result<Vec<RawChunk>> {
        // default: naive midpoint split
    }

    /// Domain-specific: merge small adjacent chunks up to budget.
    /// Return None to use kernel's default merge pass.
    /// Return Some(merged) to take full control.
    fn merge_small(
        &self,
        chunks: &[RawChunk],
        max_tokens: usize,
    ) -> Option<Vec<RawChunk>> {
        None
    }

    /// Domain-specific: what context to carry between adjacent chunks.
    /// Default: no overlap.
    fn overlap_context(
        &self,
        prev: &RawChunk,
        next: &RawChunk,
    ) -> Option<String> {
        None
    }
}
```

## ChunkingPipeline

```rust
pub struct ChunkingPipeline {
    registry: FormatRegistry,
    estimator: Arc<dyn TokenEstimator>,
    config: ChunkingConfig,
}

impl ChunkingPipeline {
    /// Process a single source file through the full pipeline.
    pub fn process(
        &self,
        source: &str,
        meta: &SourceMetadata,
    ) -> Result<Vec<ProcessedChunk>>;

    /// Batch process multiple source files.
    pub fn process_batch(
        &self,
        files: &[(String, SourceMetadata)],
    ) -> Result<Vec<ProcessedChunk>>;
}
```

### Pipeline Steps

1. **Resolve strategy**: `registry.resolve(meta.extension)` → `Arc<dyn ChunkingStrategy>`
2. **Domain chunking**: `strategy.chunk(source, meta)` → `Vec<RawChunk>`
3. **Merge small chunks**: `strategy.merge_small()` if overridden, else kernel default (merge adjacent where sum < max_tokens)
4. **Enforce token budget**: For each chunk exceeding budget, call `strategy.split_oversized()`. Recurse until all chunks fit.
5. **Add overlap context**: `strategy.overlap_context()` if overridden, else kernel default (take last N tokens of previous chunk if `overlap_tokens > 0`)
6. **Build ProcessedChunks**: Attach token estimates, processing info, strategy name.

Ordering rationale: merge before split. Merging first consolidates small chunks; splitting then handles anything still oversized. Avoids splitting a 40-token chunk that would have merged with its neighbor.

## FormatRegistry

```rust
pub struct FormatRegistry {
    defaults: HashMap<String, Arc<dyn ChunkingStrategy>>,    // kernel-provided
    overrides: HashMap<String, Arc<dyn ChunkingStrategy>>,   // adapter-provided
    fallback: Arc<dyn ChunkingStrategy>,
}

impl FormatRegistry {
    pub fn with_defaults(config: &ChunkingConfig) -> Self;
    pub fn register(&mut self, ext: &str, strategy: Arc<dyn ChunkingStrategy>);
    pub fn override_default(&mut self, ext: &str, strategy: Arc<dyn ChunkingStrategy>);
    pub fn resolve(&self, ext: &str) -> Arc<dyn ChunkingStrategy>;
}
```

Resolution priority: adapter override > adapter register > kernel default > fallback.

Kernel ships defaults: MarkdownChunker (.md), ConfigChunker (.toml/.yaml/.yml/.json), PdfChunker (.pdf), FallbackChunker (*).

## Strategy Implementations

### AstChunker (corvia-adapter-git)

**Formats:** .rs, .py, .js, .jsx, .ts, .tsx

- `chunk()`: Tree-sitter AST parse, extract top-level constructs. Filters 1-liners.
- `split_oversized()`: Recurse into child AST nodes. 500-line `impl` splits into methods. Huge functions split at inner block boundaries. Fallback: line-based split.
- `merge_small()`: Scope-aware sibling merge. Two one-liners in the same `impl` block merge. Cross-scope chunks don't. Uses `merge_group` metadata hint.
- `overlap_context()`: Prepend parent signature (e.g., `impl AuthService {` for a method chunk).

### MarkdownChunker (corvia-kernel)

**Formats:** .md

- `chunk()`: Split at heading boundaries (`#`, `##`, `###`). Each section = heading + body.
- `split_oversized()`: Recursive heading descent → paragraph splitting (`\n\n`) → sentence-level.
- `merge_small()`: Merge adjacent sections at same heading level under same parent.
- `overlap_context()`: Carry heading hierarchy (e.g., `## Auth\n### JWT\n`).

### ConfigChunker (corvia-kernel)

**Formats:** .toml, .yaml, .yml, .json

- `chunk()`: Top-level key/section chunking. Each `[section]` in TOML, each top-level key in JSON/YAML.
- `split_oversized()`: Recurse into nested keys.
- `merge_small()`: Returns None — kernel default.
- `overlap_context()`: Prepend parent key path.

### PdfChunker (corvia-kernel)

**Formats:** .pdf

- `chunk()`: Page-based extraction → paragraph splitting. Adjacent paragraphs on same page merge up to budget.
- `split_oversized()`: Sentence-level splitting within paragraph.
- `merge_small()`: Returns None — kernel default.
- `overlap_context()`: Returns None (domain adapters can override via FormatRegistry).

PDF dependency: pure Rust crate (`pdf-extract` or `lopdf`). Feature-gated if heavy.

### FallbackChunker (corvia-kernel)

**Formats:** everything not matched

- `chunk()`: Recursive separator cascade: `\n\n` → `\n` → ` ` → character-level. Produces chunks up to max_tokens.
- Other methods: all defaults (kernel handles merge/split/overlap).

## Industry Context

Approach validates against 2025-2026 industry state:
- **AST for code**: Universal (Cursor, Cody, cAST paper, SuperMemory)
- **Token budget enforcement**: Every production framework (LangChain, LlamaIndex, Chonkie, Docling)
- **Strategy/trait pattern**: Chonkie (9 strategies, one trait), Docling (template method + tokenizer interface)
- **Recursive descent for oversized**: Universal pattern across all frameworks
- **Overlap 10-20%**: Industry sweet spot, our 64/512 = 12.5%
- **Sibling merging**: Cursor and cAST both use it for chunk quality

Unique to Corvia: adapter-controlled split/merge callbacks for multi-domain support. No single-purpose framework needs this — Corvia does because it serves code, docs, config, and extensible domains.
